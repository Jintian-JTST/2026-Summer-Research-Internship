### 1. 你的代码“作弊”在了哪里？（物理上的 Bug）

你的代码其实是**直接把最终的答案（拟合公式）硬写进去了**。你人为规定了波浪的振幅是 $A=0.4$，然后让时间 $t$ 按照这个波浪去分布。

但在真实的宇宙中，物理规律不是这样运作的。作业题目明确要求：**“生成电子四动量信息...得到类似 p5 中电子数目随时间变化的图像”**。

你的代码有以下几个关键的物理缺失：

1.  **寿命用错了（相对论时间膨胀）：** 
    你用了 $\tau = 2.2 \mu s$。这是缪子在**静止状态**下的寿命。但在 Fermilab 的跑道里，缪子是以 99.9% 的光速飞行的（$\gamma \approx 29.3$）。对于实验室里的探测器来说，它的寿命被拉长了！应该是 $\tau_{lab} = \gamma \tau \approx 64.4 \mu s$。
2.  **能量生成是错的（最核心的物理）：**
    你写了 `E = np.random.rand()`，这意味着电子的能量是 0 到 1 之间均匀瞎猜的，并且**和缪子的自旋、时间毫无关系**。
    但我们在上一张幻灯片学过：**高能电子更喜欢顺着自旋方向飞**。能量 $E$ 和自旋进动是**强耦合**的！你不能直接硬点一个 $A=0.4$，真实的 $A$ 是随着你切的能量阈值（`E > 0.6`）动态变化的。
3.  **没有四动量和位置信息：**
    作业要求生成“四动量”和“位置变化”。你的代码里完全没有空间坐标 $(x, y, z)$ 和动量 $(p_x, p_y, p_z)$。

### 2. 满分作业应该怎么写？（从“数学造假”到“物理模拟”）

真正的物理 Toy MC，是不允许在代码里出现 `1 + A * np.cos(...)` 这种公式的。**波浪线（Wiggle）必须是洛伦兹变换自然而然“变”出来的！**

你需要按照宇宙的真实规律，分步骤来写：

#### 第一步：在实验室系，确定缪子的状态
*   生成一个衰变时间 $t$（用 $\tau_{lab} = 64.4 \mu s$）。
*   计算这一瞬间，缪子在跑道里的**位置**（它在做圆周运动，回旋频率 $\omega_c$）。
*   计算这一瞬间，缪子的**自旋方向**（自旋进动频率 $\omega_s$）。

#### 第二步：在缪子静止系，生成电子（用到 Michel Spectrum）
*   在缪子自己的视角里，它是个静止的胖子。
*   利用**舍选法**，根据幻灯片上的 **Michel Spectrum 公式**，生成电子的能量 $E_{rest}$ 和出射角度 $\vartheta$（相对于自旋方向的夹角）。
*   **构建四动量：** 把能量和角度写成一个四维向量 $P_{rest} = (E_{rest}/c, p_x, p_y, p_z)$。

#### 第三步：见证奇迹的时刻（洛伦兹变换 Boost）
*   把静止系下的电子四动量 $P_{rest}$，沿着缪子在实验室里的飞行方向，进行**洛伦兹变换**，得到实验室系下的四动量 $P_{lab}$。
*   从 $P_{lab}$ 中提取出电子在实验室里的真实能量 $E_{lab}$。

#### 第四步：筛选与画图
*   对 $E_{lab}$ 设定阈值（比如 `if Elab > 1.7 GeV:`）。
*   把满足条件的 $t$ 画成直方图。**你会震惊地发现，虽然你的代码里没有任何 `cos` 波浪公式，但画出来的图自然而然地出现了完美的 Wiggle 波浪！** 这就是物理学的魅力！

### 3. 导师给你的一点“代码外挂”

我知道大一新生写“四动量”和“洛伦兹变换”会很痛苦。我给你提供一个 Python 里做洛伦兹变换的数学工具函数，你可以直接用到你的作业里：

```python
import numpy as np

def lorentz_boost(E_rest, p_rest_vector, beta_vector):
    """
    将静止系下的四动量 Boost 到实验室系
    E_rest: 静止系下的能量
    p_rest_vector: 静止系下的三维动量 numpy array，例如 np.array([px, py, pz])
    beta_vector: 缪子在实验室系下的速度向量 (v/c) numpy array，例如 np.array([beta_x, beta_y, beta_z])
    返回: 实验室系下的能量 E_lab
    """
    beta2 = np.dot(beta_vector, beta_vector)
    if beta2 == 0:
        return E_rest
    
    gamma = 1.0 / np.sqrt(1.0 - beta2)
    beta_dot_p = np.dot(beta_vector, p_rest_vector)
    
    # 洛伦兹变换公式 (只算能量部分)
    E_lab = gamma * (E_rest + beta_dot_p) 
    # 如果你需要实验室系的三维动量，公式是：
    # p_lab_vector = p_rest_vector + (gamma - 1) * beta_dot_p / beta2 * beta_vector + gamma * E_rest * beta_vector
    
    return E_lab
```
